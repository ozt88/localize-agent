- `parent_choice_text` 컬럼이 스캔에 포함되도록 row scan 수정.

**1-3. clustertranslate/types.go** — `ClusterTask` 구조체에 6개 필드 추가:
```go

PrevKO           []string          // CONT-02
NextKO           []string          // CONT-02
VoiceCards       map[string]string // TONE-02
ParentChoiceText string            // BRANCH-01
RAGHints         string            // D-17
```
소스: git 71f1c64 참조.

**1-4. clustertranslate/voice_card.go** — `VoiceCard` 구조체에 `Relationships map[string]string` 필드 추가 (`json:"relationships,omitempty"`). `LoadVoiceCards` 함수는 기존대로 JSON 디코딩하므로 자동 호환.

### Task 3: voice_cards.json 재생성 (wiki 기반 + Kattegatt + relationships)

**Files:**
- `workflow/cmd/go-generate-voice-cards/main.go` (MODIFY)
- `projects/esoteric-ebb/context/voice_cards.json` (CREATE)

**Action:** MODIFY/CREATE

**3-1. go-generate-voice-cards CLI 개선:**
- `--wiki-dir` 플래그 추가 (wiki_markdown 디렉토리 경로)
- 입력 소스를 3가지로 확장: (a) wiki 캐릭터 페이지 텍스트, (b) DB 공동 출현 화자, (c) DB 대화 샘플
- wiki 매칭: `speaker_allow_list.json`의 캐릭터 이름으로 `wiki_markdown/` 디렉토리 파일 탐색. 파일명에 캐릭터 이름이 포함되면 매칭 (case-insensitive contains). 매칭 실패 시 해당 캐릭터는 wiki 없이 진행.
- LLM 프롬프트에 relationships 생성 지시 추가: "화자가 다른 캐릭터와 대화할 때 어조/존댓말 변화를 relationships로 출력"
- 출력 JSON 형식: `{ "Speaker": { "speech_style": "...", "honorific": "...", "personality": "...", "relationships": { "Other": "..." } } }`
- speech_style에 "어미: ~야, ~잖아" 패턴 구체적 어미 예시 포함 지시
- Kattegatt 포함: speaker_allow_list.json에 없으면 하드코딩으로 추가. 고어체 프로필: `speech_style: "고어체. 어미: ~도다, ~노라, ~옵니다. thou/thy → 그대/당신. 장중하고 느린 리듬"`, `honorific: "존대 (모든 대상에게 고어 격식체)"`, `personality: "위엄 있는 고대 존재, 신탁적, 신비로운"`, relationships default: "초월적 거리감 — 모두에게 고어체".

**3-2. CLI 실행:**
```bash
go run ./workflow/cmd/go-generate-voice-cards \
  --project projects/esoteric-ebb/project.json \
  --wiki-dir projects/esoteric-ebb/rag/wiki_markdown/ \
  --output projects/esoteric-ebb/context/voice_cards.json \
  --backend postgres
```

**3-3. 출력 검증:**
- `voice_cards.json` 존재 확인
- Kattegatt 키 존재 + `고어체` / `그대` 포함 확인
- 16명 이상(15 기존 + Kattegatt) 엔트리 확인
- 각 엔트리에 `speech_style`, `honorific`, `personality`, `relationships` 4개 필드 존재
