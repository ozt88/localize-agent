### Task 1: 전체 DB 리셋 + 파이프라인 실행

**Files:** (코드 변경 없음 — CLI 실행만)

**Action:** 실행

**1-1. 전체 리셋:**
```bash
go run ./workflow/cmd/go-v2-reset-all \
  --project projects/esoteric-ebb/project.json \
  --backend postgres
```
per knowledge/guardrails.md: `--backend postgres` 명시 필수.

출력에서 `total=35009` (또는 유사 수치) 확인. total이 0이면 중단 — `--backend postgres` 누락일 가능성 (knowledge/troubleshooting.md: "total=0 버그").

**1-2. 실행 전 체크리스트 (per knowledge/guardrails.md):**
```bash
# 플래그 확인
go run ./workflow/cmd/go-v2-pipeline --help 2>&1 | grep -E "voice-cards|rag-context"
# 데이터 파일 확인
ls -la projects/esoteric-ebb/context/voice_cards.json
ls -la projects/esoteric-ebb/rag/rag_batch_context.json
```

**1-3. 파이프라인 실행:**
```bash
go run ./workflow/cmd/go-v2-pipeline \
  --project projects/esoteric-ebb/project.json \
  --backend postgres \
  --voice-cards projects/esoteric-ebb/context/voice_cards.json \
  --rag-context projects/esoteric-ebb/rag/rag_batch_context.json \
  --translate-concurrency 8 \
  --cleanup-stale-claims
```

장시간 실행 (35K건, 예상 수 시간~하루). 별도 터미널에서 진행 상황 모니터링:
```sql
SELECT state, COUNT(*) FROM pipeline_items_v2 GROUP BY state ORDER BY state;
```

**1-4. 실행 중 장애 대응:**
- OpenCode 서버 hang: `--cleanup-stale-claims` 플래그로 stale lease 자동 회수
- watchdog false-kill: Phase 08에서 probeServer로 수정 완료 (knowledge/decisions.md)
- warmup 실패: UpdateRetryState 구현 완료 (knowledge/guardrails.md)

**Verification:**
```sql

-- done 상태가 35,009건(또는 그에 근접)이어야 한다
-- pending_translate = 0, working_translate = 0 확인
```

### Task 2: 번역 완료 통계 + 품질 스팟체크

**2-1. 완료 통계:**
