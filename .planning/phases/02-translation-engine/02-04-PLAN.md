Create `workflow/internal/v2pipeline/worker.go` with three worker functions. Each follows the pattern: claim batch -> process -> mark result -> append attempt log.

**Session key convention per Pitfall 3:** `v2-translate-{workerID}`, `v2-format-{workerID}`, `v2-score-{workerID}`

**TranslateWorker:**
```go
func TranslateWorker(ctx context.Context, cfg Config, store contracts.V2PipelineStore, llm *platform.SessionLLMClient, glossarySet *glossary.GlossarySet, workerID string) error
```
Loop:
1. `store.ClaimPending(StatePendingTranslate, StateWorkingTranslate, workerID, cfg.TranslateBatchSize, cfg.LeaseSec)`
2. If no items, sleep cfg.IdleSleepSec, continue
3. Group claimed items by batch_id (items in same batch should be translated together for scene context per TRANS-01)
4. For each batch group:
   a. Build `ClusterTask` from items (look up batch info, previous gate context per D-03)
   b. Warmup: `llm.EnsureContext("v2-translate-"+workerID, translateProfile)` with base warmup + glossary warmup terms (D-11)
   c. Build prompt: `clustertranslate.BuildScriptPrompt(task)`
   d. Send: `llm.SendPrompt("v2-translate-"+workerID, prompt)`
   e. Validate: `clustertranslate.ValidateTranslation(rawOutput, meta, sourceTexts)`
   f. If validation fails:
      - Check attempt count per D-15: if item.TranslateAttempts < 2, retry with hint (append score reason if available from previous attempt_log)
      - If attempts >= 2, escalate to high model (use cfg high_llm profile)
      - If attempts >= 3 (max_retries), `store.MarkFailed(id, error)`
      - `store.AppendAttemptLog(id, {attempt, stage:"translate", model, failure_type:nil, reason:err.Error(), timestamp})`
   g. If validation passes:
      - Parse: `clustertranslate.ParseNumberedOutput(rawOutput)` then `MapLinesToIDs`
      - For each mapped item: `store.MarkTranslated(id, koRaw)` (this auto-routes to pending_format or pending_score based on has_tags)
      - `store.AppendAttemptLog(id, {attempt, stage:"translate", model, failure_type:nil, reason:nil, timestamp})`

**FormatWorker:**
```go
func FormatWorker(ctx context.Context, cfg Config, store contracts.V2PipelineStore, llm *platform.SessionLLMClient, workerID string) error
```
Loop:
1. `store.ClaimPending(StatePendingFormat, StateWorkingFormat, workerID, cfg.FormatBatchSize, cfg.LeaseSec)`
2. If no items, sleep, continue
3. For each small batch (3-5 items per D-06):
   a. Build `[]FormatTask` from items (ENSource=item.SourceRaw, KOPlain=item.KORaw)
   b. Warmup: `llm.EnsureContext("v2-format-"+workerID, formatProfile)`
   c. Send: `llm.SendPrompt("v2-format-"+workerID, tagformat.BuildFormatPrompt(tasks))`
   d. Parse: `tagformat.ParseFormatResponse(raw, len(tasks))`
   e. Validate each: `tagformat.ValidateTagMatch(item.SourceRaw, koFormatted)` per D-07/TRANS-06
   f. If validation fails per D-08:
      - FormatAttempts < 2: retry with hint (include tag mismatch details)
      - FormatAttempts == 2: escalate to gpt-5.4 (high_llm profile)
      - FormatAttempts >= 3: `store.MarkFailed(id, error)`
   g. If passes: `store.MarkFormatted(id, koFormatted)`
   h. Always: `store.AppendAttemptLog(id, {attempt, stage:"format", model, ...})`

**ScoreWorker:**
```go
func ScoreWorker(ctx context.Context, cfg Config, store contracts.V2PipelineStore, llm *platform.SessionLLMClient, workerID string) error
```
Loop:
1. `store.ClaimPending(StatePendingScore, StateWorkingScore, workerID, cfg.ScoreBatchSize, cfg.LeaseSec)`
2. If no items, sleep, continue
3. For each item (single-item scoring per Open Question 2 in RESEARCH.md):
   a. Build `ScoreTask` from item
   b. Warmup: `llm.EnsureContext("v2-score-"+workerID, scoreProfile)`
   c. Send: `llm.SendPrompt("v2-score-"+workerID, scorellm.BuildScorePrompt(task))`
   d. Parse: `scorellm.ParseScoreResponse(raw)` per Pitfall 5 (handle invalid JSON)
   e. If parse error: treat as score error, append to attempt_log, retry (not a translation failure)
   f. If parsed: `store.MarkScored(id, result.ScoreFinal(), result.FailureType, result.Reason)` -- this auto-routes per D-14
   g. `store.AppendAttemptLog(id, {attempt, stage:"score", model, failure_type:result.FailureType, reason:result.Reason, score:result.TranslationScore, timestamp})`

Create `workflow/internal/v2pipeline/run.go`:

```go
func Run(cfg Config) int
```

Flow:
1. Open store: `OpenStore(cfg.CheckpointBackend, "", cfg.CheckpointDSN)`
2. If cfg has CleanupStaleClaims: run `store.CleanupStaleClaims(cfg.LeaseSec * 3)`
3. Print initial state: `store.CountByState()`
4. Load glossary: `glossary.LoadGlossary(glossaryPath, locTextsDir, speakerNames)`
5. Create LLM client: `platform.NewSessionLLMClient(cfg.TranslateServerURL, cfg.TranslateTimeoutSec, metrics, traceSink)`
6. Build LLM profiles for 3 roles:
   - translate: {ProviderID, ModelID from cfg.TranslateModel, Agent: "v2-translate", Warmup: BuildBaseWarmup with v2_base_prompt + glossary warmup}
   - format: {ModelID from cfg.FormatModel, Agent: "v2-format", Warmup: BuildFormatWarmup from v2_format_prompt}
   - score: {ModelID from cfg.ScoreModel, Agent: "v2-score", Warmup: BuildScoreWarmup from v2_score_prompt}
7. Launch worker goroutines based on cfg concurrency:
   - cfg.TranslateConcurrency x TranslateWorker
   - cfg.FormatConcurrency x FormatWorker
   - cfg.ScoreConcurrency x ScoreWorker
8. Wait for ctx cancellation (SIGINT/SIGTERM) or all workers to return
9. Print final state: `store.CountByState()`
10. Return exit code
