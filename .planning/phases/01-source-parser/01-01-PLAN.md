<ink_json_structure>
<!-- Key ink JSON patterns from actual TextAsset files -->

Root structure: {"inkVersion":21, "root":[...], "listDefs":{}}

Text content: strings starting with "^" (e.g., "^You approach the half-sphere. ")
Newline: "\n" string literal
Tags: "#" opens tag, "/#" closes tag (e.g., "#", "^speaker ", "/#")
Evaluation: "ev" opens, "/ev" closes (SKIP these)
String building: "str" opens, "/str" closes (SKIP these)
Divert: {"->":"knot_name"} or {"->":"relative.path"}
Choice: {"*":".^.^.c-N", "flg":N} where flg is bitfield
Gate container: {"#n":"g-N"} in last-element metadata dict
Choice container: {"#n":"c-N"} in last-element metadata dict
Knot: top-level key in root object (e.g., "CoastMap")
Glue: "<>" string literal (merge text across divert, 10 files, 34 occurrences)

Container last element: if dict, contains metadata (#f, #n) and sub-containers (g-N, c-N)
If null: empty metadata.

flg bitfield (D-26):
  0x1  = conditional
  0x2  = has start content
  0x4  = choice-only content
  0x8  = invisible default
  0x10 = once-only
</ink_json_structure>

    **Step 0 — Create types.go (per D-05, D-20, D-21):**

    ```go
    package inkparse

    // DialogueBlock represents one game-rendering unit of text.
    // Consecutive "^text" entries within a gate/choice container are merged into one block.
    type DialogueBlock struct {
        ID         string   `json:"id"`          // path-based: "KnotName/g-0/c-1/blk-0"
        Path       string   `json:"path"`        // tree path without blk suffix
        Text       string   `json:"text"`        // merged text content (^ prefix stripped)
        SourceHash string   `json:"source_hash"` // SHA-256 hex of Text
        SourceFile string   `json:"source_file"` // TextAsset filename (e.g., "AR_CoastMap")
        Knot       string   `json:"knot"`        // knot name
        Gate       string   `json:"gate"`        // gate ID (e.g., "g-0") or empty
        Choice     string   `json:"choice"`      // choice ID (e.g., "c-1") or empty
        Speaker    string   `json:"speaker"`     // from # speaker tag, empty if none
        Tags       []string `json:"tags"`        // DC_check, OBJ, XPGain, etc.
        BlockIndex int      `json:"block_index"` // sequential index within container
    }

    // ParseResult holds all blocks extracted from one TextAsset file.
    type ParseResult struct {
        SourceFile       string           `json:"source_file"`
        Blocks           []DialogueBlock  `json:"blocks"`
        TotalTextEntries int              `json:"total_text_entries"`
    }
    ```

    **Step 1 — Create hash.go (per PREP-01):**

    ```go
    package inkparse

    import (
        "crypto/sha256"
        "encoding/hex"
    )

    // SourceHash returns the hex-encoded SHA-256 of text.
    func SourceHash(text string) string {
        h := sha256.Sum256([]byte(text))
        return hex.EncodeToString(h[:])
    }
    ```

    **Step 2 — Write RED tests in parser_test.go** for all behaviors listed above, using small inline JSON fixtures that match the ink JSON structure observed in AR_CoastMap.txt. Test fixtures should be minimal (3-5 element arrays, not full files).

    **Step 3 — Implement parser.go (GREEN):**

    Core function `Parse(data []byte, sourceFile string) (*ParseResult, error)`:
    1. Unmarshal JSON into `map[string]any`
    2. Extract "root" array
    3. For each top-level key in root (knots): recursively walk the container
    4. Walking logic per D-20 through D-25:
       - If element is string starting with "^": strip "^", accumulate into current text block
       - If element is "\n": append newline to current text (does NOT end block)
       - If element is "#": next string is tag content, until "/#"
       - If element is "ev": skip until "/ev"
       - If element is "str": skip until "/str"
       - If element is "<>": set glue flag (per D-16, D-17)
       - If element is dict with "->": end current block, record divert target
       - If element is dict with "*" and "flg": parse choice, extract text from "s" sub-container
       - If last element of array is dict: check for "#n" (container name g-N/c-N), recurse into sub-containers
       - If last element is null: no metadata
    5. Block ID generation (per D-02 discretion: path-based): `{Knot}/{Gate}/{Choice}/blk-{N}`
    6. After accumulating text, call `SourceHash(text)` for each block

    **Step 4 — Implement glue.go** for the 34 glue markers across 10 files (per D-18, D-19):
    - When "<>" is encountered, set a flag that the next text (even across a divert) should be joined without newline to the previous block's text
    - `resolveGlue(blocks []DialogueBlock, gluePoints []GluePoint) []DialogueBlock`

    **Step 5 — REFACTOR:** Clean up, ensure all tests pass.

    Important: Do NOT simulate the ink runtime (per D-23). Walk arrays sequentially. Only follow diverts for glue resolution (per D-19).
