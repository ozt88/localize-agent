Capture data format (full_text_capture_clean.json):
```json
{
  "count": 4550,
  "entries": [
    {
      "text": "You approach the half-sphere. ",
      "origin": "ink_dialogue",   // or "ink_choice", "menu_scan", "tmp_text"
      "has_tags": false,
      "length": 29,
      "dialog_source": ""
    }
  ]
}
```

Origin types:
- "ink_dialogue": text that came through ink dialogue system (COMPARE these)
- "ink_choice": choice text from ink (COMPARE these)
- "menu_scan": UI/menu text (NOT from ink parser, skip)
- "tmp_text": TextMeshPro text (NOT from ink parser, skip)
</interfaces>

<validation_strategy>
<!-- From D-06, D-07, D-08 -->

Validation approach:
1. Filter capture entries to origin="ink_dialogue" and origin="ink_choice" only
2. For each capture entry, strip rendering wrappers (per D-07): <line-indent=*>, <#hex>, <size=*>, <smallcaps>, </line-indent>, </size>, </smallcaps>, closing color tags
3. Normalize whitespace (trim, collapse internal whitespace)
4. Look for matching text in parser output (exact match after normalization)
5. Report: total capture entries, matched, unmatched, match rate %
6. Log each unmatched entry with its text (truncated) for debugging

Target: 95%+ match rate on ink_dialogue + ink_choice entries
</validation_strategy>

    **Step 1 — Define types in validate.go:**

    ```go
    package inkparse

    // CaptureEntry represents one entry from full_text_capture_clean.json
    type CaptureEntry struct {
        Text         string `json:"text"`
        Origin       string `json:"origin"`
        HasTags      bool   `json:"has_tags"`
        Length       int    `json:"length"`
        DialogSource string `json:"dialog_source"`
    }

    // CaptureData is the root structure of the capture JSON file
    type CaptureData struct {
        Count   int            `json:"count"`
        Entries []CaptureEntry `json:"entries"`
    }

    // ValidationReport holds results of comparing parser output to capture data
    type ValidationReport struct {
        TotalCapture    int              `json:"total_capture"`     // ink_dialogue + ink_choice entries
        Matched         int              `json:"matched"`
        Unmatched       int              `json:"unmatched"`
        MatchRate       float64          `json:"match_rate"`        // 0.0 to 1.0
        UnmatchedItems  []UnmatchedItem  `json:"unmatched_items"`
        SkippedOrigins  map[string]int   `json:"skipped_origins"`  // menu_scan, tmp_text counts
    }

    type UnmatchedItem struct {
        Text   string `json:"text"`
        Origin string `json:"origin"`
    }
    ```

    **Step 2 — Implement normalizeForComparison:**

    Strip rendering wrappers added by the game engine (per D-07):
    - `<line-indent=[^>]*>` and `</line-indent>`
    - `<#[0-9A-Fa-f]{6,8}>` (hex color wrappers)
    - `<size=[^>]*>` and `</size>`
    - `<smallcaps>` and `</smallcaps>`
    - Trim whitespace, collapse consecutive whitespace to single space

    Preserve content tags: `<b>`, `</b>`, `<i>`, `</i>`, `<color=[^>]*>`, `</color>` -- these are part of the ink content.

    **Step 3 — Implement ValidateAgainstCapture:**

    ```go
    func ValidateAgainstCapture(blocks []DialogueBlock, captureData CaptureData) ValidationReport
    ```

    1. Build a set of normalized parser block texts (from all blocks)
    2. Filter capture entries to origin=="ink_dialogue" or origin=="ink_choice"
    3. For each filtered entry: normalize, check if present in block text set
    4. Record matched/unmatched counts, collect unmatched items
    5. Calculate match rate as matched/total

    Also provide:
