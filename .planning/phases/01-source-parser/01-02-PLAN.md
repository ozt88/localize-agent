<objective>
Classify parsed dialogue blocks by content type (dialogue/spell/UI/item/system), detect passthrough strings that don't need translation, and group blocks into content-type-optimized translation batches.

Purpose: Different content types need different LLM prompt formats and batch sizes for optimal translation quality. Passthrough detection prevents wasting LLM calls on untranslatable strings.
Output: classifier, passthrough detector, and batcher in `workflow/internal/inkparse/`
</objective>

    ```go
    // Batch represents a group of blocks ready for translation.

        ID          string           `json:"id"`           // e.g., "SourceFile/knot/g-0"
        ContentType string           `json:"content_type"` // dialogue, spell, ui, item, system
        Format      string           `json:"format"`       // script, card, dictionary, document
        Blocks      []DialogueBlock  `json:"blocks"`
        SourceFile  string           `json:"source_file"`
        Knot        string           `json:"knot"`
        Gate        string           `json:"gate"`         // gate = cluster boundary per D-22
    }
    ```

    Function:

    ```go
    // BuildBatches groups blocks into translation batches by content type and gate boundary.
    // Passthrough blocks are excluded. Gate boundaries define cluster edges for dialogue.
    func BuildBatches(results []ParseResult) []Batch
    ```

    Logic per content type:
    - **dialogue** (format="script"): Group by gate (per D-22). Each gate = one batch. If gate has >30 blocks, split at 30. If gate has <10 blocks, try to merge with adjacent gate in same knot.
    - **spell** (format="card"): Group 5-10 items per batch.
    - **ui** (format="dictionary"): Group 50-100 items per batch.
    - **item** (format="card"): Group 5-10 items per batch.
    - **system** (format="document"): Entire section as one batch.

    Knot entry (blocks before first gate) gets its own batch (per D-22).

    Exclude blocks where `IsPassthrough == true`.

    **Tests in batcher_test.go:**
    - 20 dialogue blocks in same gate -> 1 batch, format="script"
    - 40 dialogue blocks in same gate -> 2 batches (split at 30)
    - 5 spell blocks -> 1 batch, format="card"
    - 80 ui blocks -> 1 batch, format="dictionary"
    - Passthrough blocks excluded from all batches
    - Blocks from different gates -> different batches
