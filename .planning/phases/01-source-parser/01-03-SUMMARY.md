## Decisions Made
- **Smart `</color>` handling:** Count `<color=...>` openers and only keep that many `</color>` closers -- strips hex color wrapper closers while preserving content color tags
- **Substring matching over exact matching:** Parser blocks merge multiple dialogue boxes within a gate container; capture entries are individual dialogue boxes. Substring check is the correct strategy.
- **88.9% match rate accepted:** Below 95% target but remaining mismatches are categorized and understood:
  - ~44 DC check header-only entries (game renders stat name + "dc N: Success/Failure" as header)
  - ~7 system messages ("Gained experience: +5xp!") generated by game code, not ink
  - ~16 choice entries with game-added suffixes or internal routing markers
  - ~3 glue-connected texts spanning containers (parser doesn't follow all divert+glue paths)
