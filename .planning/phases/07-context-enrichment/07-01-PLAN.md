per D-01/D-02/D-03, workflow/cmd/go-generate-voice-cards/main.go 생성:

CLI 플래그:
- `-project` (string): project.json 경로 (기본: projects/esoteric-ebb/project.json)
- `-output` (string): 출력 voice_cards.json 경로 (기본: projects/esoteric-ebb/context/voice_cards.json)
- `-speakers` (string): speaker_allow_list.json 경로 (기본: projects/esoteric-ebb/context/speaker_allow_list.json)
- `-sample-count` (int): 캐릭터당 대사 샘플 수 (기본: 20)
- `-min-frequency` (int): voice card 생성 최소 빈도 (기본: 100, D-03에 따라 Thal 100까지)

동작 흐름:
1. speaker_allow_list.json 로드, frequency >= min-frequency인 named 캐릭터 필터 (ability-score speaker wis/str/int/cha/dex/con 제외)
2. project.json 로드 -> DB 연결 (checkpoint_dsn 또는 checkpoint_db)
3. 각 캐릭터별로 DB에서 대사 샘플 추출:
   ```sql
   SELECT source_raw FROM pipeline_items_v2
   WHERE speaker = ? AND state = 'done'
   ORDER BY RANDOM() LIMIT ?
   ```
4. LLM에 캐릭터별 프롬프트 전송:
   ```
   다음은 게임 캐릭터 "{name}"의 영어 대사 샘플입니다:
   {samples}

   이 캐릭터의 기본 모드(baseline tone)를 분석하여 한국어 번역 시 적용할 voice profile을 JSON으로 작성하세요:
   - speech_style: 이 캐릭터의 화법 스타일 (예: "조용하고 신중한 어조, 짧은 문장 선호")
   - honorific: 존댓말 레벨 ("반말", "평어", 또는 "존대" 중 하나)
   - personality: 성격 키워드 2-3개 (예: "내성적, 관찰력 있음, 가끔 날카로운 유머")

   JSON만 출력하세요: {"speech_style":"...", "honorific":"...", "personality":"..."}
   ```
5. LLM 응답에서 JSON 파싱 -> map[string]VoiceCard 조합
6. json.MarshalIndent로 output 경로에 저장

실행 후 생성된 voice_cards.json을 git에 커밋. 이 파일은 데이터 자산이므로 버전 관리 대상.

**주의사항:**
- LLM 호출 실패 시 해당 캐릭터 스킵하고 stderr에 경고 출력
- 이미 output 파일이 존재하면 기존 데이터를 로드하고, 누락된 캐릭터만 추가 생성 (idempotent)
- ability-score speaker(wis/str/int/cha/dex/con)는 반드시 제외 (이미 abilityScoreVoice 맵에 별도 관리)
