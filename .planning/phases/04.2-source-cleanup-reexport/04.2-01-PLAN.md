    **Create `workflow/cmd/go-v2-source-cleanup/main.go`** -- a one-shot CLI tool per D-04, D-05, D-07.

    The tool does:

    1. **Parse flags:** `--project` (path to project.json), `--parse-result` (path to parse_result.json from re-running the parser), `--dry-run` (default true, set false to actually write), `--dsn` (PostgreSQL DSN, default from project.json checkpoint_dsn).

    2. **Load parse_result.json:** Read the JSON file containing `[]ParseResult` (array of per-file results). Flatten all blocks into a map keyed by block ID.

    3. **Connect to PostgreSQL** using `pgx/v5/stdlib` (same pattern as store.go).

    4. **Identify changed blocks:** Query DB:
    ```sql
    SELECT id, source_raw, source_hash, ko_raw, ko_formatted
    FROM pipeline_items_v2
    WHERE id = ANY($1)
    ```
    For each block where DB source_raw != parse_result source_raw (i.e., DC/FC was stripped), collect as a "changed" record. Expected ~579 records.

    5. **Create snapshot table (per D-05):**
    ```sql
    CREATE TABLE IF NOT EXISTS source_cleanup_snapshot (
        id TEXT PRIMARY KEY,
        old_source_raw TEXT NOT NULL,
        old_source_hash TEXT NOT NULL,
        old_ko_raw TEXT,
        old_ko_formatted TEXT,
        created_at TEXT NOT NULL
    );
    ```

    6. **UNIQUE collision check (per D-04):** For each changed block, compute new source_hash = SHA-256 hex of new source_raw. Then:
    ```sql
    SELECT id, source_raw FROM pipeline_items_v2
    WHERE source_hash = $1 AND id != $2
    ```
    If any collisions found, print them and ABORT (do not update those rows). Log collision count.

    7. **Execute in transaction (if not dry-run):**
    a) INSERT snapshot rows
    b) UPDATE source_raw, source_hash for changed blocks
    c) For ko_raw and ko_formatted: apply `^[A-Z]{2}\d+\s+\w+-` regex strip (same as dcfcPrefixRe). Use Go regex, not SQL regex. Read current ko values, strip prefix if present, UPDATE back.

    8. **Print summary:** total changed, total collisions skipped, total ko_raw stripped, total ko_formatted stripped.

    **Then execute the migration:**

    a) First, re-run the parser to generate fresh parse_result.json:
    ```
    go run ./projects/esoteric-ebb/cmd/go-esoteric-adapt-in --project projects/esoteric-ebb/project.json
    ```
    (This produces the parse_result.json with DC/FC-stripped sources)

    b) Run the cleanup tool in dry-run mode first:
    ```
    go run ./workflow/cmd/go-v2-source-cleanup --project projects/esoteric-ebb/project.json --parse-result <path-to-parse-result.json> --dry-run
    ```

    c) Verify dry-run output shows ~579 changes, 0 collisions.

    d) Run for real:
    ```
    go run ./workflow/cmd/go-v2-source-cleanup --project projects/esoteric-ebb/project.json --parse-result <path-to-parse-result.json> --dry-run=false
    ```

    Use `crypto/sha256` + `encoding/hex` for source hash (same as inkparse.SourceHash).
    Use `regexp.MustCompile("^[A-Z]{2}\\d+\\s+\\w+-")` for ko_raw/ko_formatted strip.
    Import `database/sql`, `github.com/jackc/pgx/v5/stdlib`, `localize-agent/workflow/pkg/shared`.
