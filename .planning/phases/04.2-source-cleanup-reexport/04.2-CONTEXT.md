<decisions>
## Implementation Decisions

### 파서 DC/FC strip (D-01)
- **D-01:** 파서(parser.go)에서 DC/FC 접두사(`DC12 str-`, `FC8 int-` 등)를 source_raw에서 strip. 능력치 타입(`str`, `int`, `wis` 등)은 tags[] 배열에 메타데이터로 보존.
  - 예: `"FC8 int-<i>Ragn?</i>"` → source_raw=`"<i>Ragn?</i>"`, tags=[..., "FC8", "int"]
  - DC/FC 번호는 이미 tags[]에 존재 (예: "DC8", "FC10")
  - 능력치 타입은 기존 tags[]에 없으므로 새로 추가

### passthrough 변경 없음 (D-02)
- **D-02:** passthrough 로직은 변경하지 않음. 미번역 3,130건 중 80%가 렌더링 래퍼 문제(Plugin.cs 영역), passthrough 확장 효과는 142건(숫자/레벨)뿐. 별도 작업으로 분리.

### 파이프라인 단순화 (D-03)
- **D-03:** 핵심 가치: 파서에서 clean source 생성 → export.go DC/FC 우회 로직 제거 → Plugin.cs Stage 2b(DC/FC strip 매칭) 제거.
  - export.go: DC/FC body-only entry 생성 로직(108-132행) 제거
  - export.go: CleanTarget()의 dcfcPrefixRe strip(145행) 제거
  - Plugin.cs: TryTranslate 4→3단계 (TranslationMap → Contextual → RuntimeLexicon)
  - export.go 유지 항목: ko_raw fallback, NormalizeLLMEscapes, 줄 분해, abilityPrefixRe

### DB UPDATE in-place (D-04)
- **D-04:** 파서 재실행 후 DC/FC 접두사가 있었던 ~579건만 source_raw/source_hash를 UPDATE. 나머지 ~34,450건은 변경 없음.
  - 블록 ID 기반 매칭 (같은 ink JSON이므로 동일 ID 생성 보장)
  - 실행 전 UNIQUE 충돌 체크 쿼리 필수 (clean source_hash가 기존 항목과 겹치지 않는지)

### 스냅샷 백업 (D-05)
- **D-05:** UPDATE 전 579건의 기존 source_raw/source_hash를 스냅샷 테이블에 저장. 경량 롤백 지원.

### 전량 재생성 (D-06)
- **D-06:** DB UPDATE 후 전체 패치 아티팩트 재생성: translations.json + TextAsset(286 파일) + localizationtexts.csv + runtime_lexicon.json.

### ko_raw 정리 (D-07)
- **D-07:** 579건의 ko_raw/ko_formatted에서도 DC/FC 접두사 제거. source와 target 모두 깨끗하게. DB 자체가 clean 상태.

### export.go 정리 (D-08)
- **D-08:** 제거: DC/FC body-only entry 생성 로직 + CleanTarget의 dcfcPrefixRe strip. 유지: ko_raw fallback, NormalizeLLMEscapes, abilityPrefixRe strip, 줄 분해(explode).

### Plugin.cs 3단계 (D-09)
- **D-09:** Stage 2b(DC/FC strip 매칭) 제거. TryTranslate: TranslationMap → Contextual → RuntimeLexicon (3단계). source가 clean하므로 DC/FC strip 불필요.

### 인게임 검증 포함 (D-10)
- **D-10:** 패치 재배포 → 게임 실행 → DC/FC 선택지 정상 표시 확인 → translations_loaded ≥ 75,789 회귀 없음 → hit rate 확인.

### Claude's Discretion
- 파서 DC/FC strip 구현 위치 (walkFlatContent vs 별도 함수)
- 스냅샷 테이블 스키마/이름
- 충돌 체크 스크립트 구현 방식
- 검증용 씬 선정

</decisions>
