<decisions>
## Implementation Decisions

### 재작성 전략
- **D-01:** 완전 신규 작성. 빈 파일에서 시작하여 v1 코드를 참고하되 필요한 기능만 선별적으로 가져온다. 3,083줄 → 1,500~2,000줄 목표.
- **D-02:** 단일 파일(Plugin.cs) 유지. BepInEx 플러그인 특성상 단일 파일이 배포 간편.

### 데이터 구조
- **D-03:** NormalizedMap 완전 제거. v2에서는 정확한 소스 매칭만 사용, normalize 폴백 없음.
- **D-04:** TranslationMap(exact match) + ContextualMap(source_file 기반 동음이의 해소) 2개 사전만 사용.

### TryTranslate 체인 (확정, ROADMAP SC-3)
- **D-05:** 4단계 순서: TranslationMap → DC/FC strip 매칭 → Contextual → RuntimeLexicon. (참고: 원래 5개 항목으로 기술되었으나, RESEARCH Open Question 2 결론에 따라 GeneratedPattern을 RuntimeLexicon regex_rules로 통합하여 실질 4단계로 확정.)
- **D-06:** StripQuotationMarks 없음. CleanOrphanBoldTags는 orphan-only(균형 <b>는 유지).

### 하드코딩 번역 이동
- **D-07:** 29건 전부 runtime_lexicon.json으로 이동. Plugin.cs에 하드코딩 번역 0건.
  - MenuDirectOverrides 8건 → exact 타입
  - StatNameOverrides 6건 → regex 타입 (\bStrength\b 등)
  - TranslateShortSystemPhrase 8건 → exact 또는 substring 타입
  - TranslateRichSystemFragments 7건 → regex 타입
- **D-08:** runtime_lexicon.json은 exact + substring + regex 3종 규칙 타입 지원 (v1에서 이미 구현된 형식 재사용).

### Harmony 패치 구조
- **D-09:** 확실히 필요한 6개 패치는 무조건 포함:
  - TextAssetTextPostfix (ink JSON 교체)
  - TmpTextPrefix (UI/메뉴/시스템 텍스트)
  - DialogAddChoiceTextPrefix (선택지 번역)
  - LocalizeCheckLanguagePostfix (I2 로컬라이제이션)
  - TmpOnEnablePostfix + TmpConcreteAwakePostfix (한국어 폰트 폴백 주입)
- **D-10:** 불확실한 7개 패치는 연구 단계에서 게임 런타임 분석으로 필요성 판단:
  - DialogAddTextPrefix, DialogStartDialogPrefix
  - UiTextPrefix, UiElementsTextPrefix, UiOnEnablePostfix
  - MenuController 3개 (Start/Refresh/Update)
  - SceneLoadedPostfix

### 폰트 로딩
- **D-11:** TryLoadKoreanFont를 기존 코드를 학습하면서 새로 작성. Regular 우선 로드. 핵심 로직만 추출하여 간소화.

### 캡처/디버깅 인프라
- **D-12:** 핵심 3개만 유지:
  - untranslated_capture.json — 항상 ON (미번역 문자열 자동 수집)
  - translation_hits.json — ENABLE_FULL_CAPTURE 시만 (source→target 매핑 로그)
  - translation_loader_state.json — 항상 ON (hit/miss 카운터)
- **D-13:** 제거 대상: full_text_capture, choice_capture, menu_runtime_dump, ui_toolkit_dump

### 선택지 처리 (확정, Phase 04 Bug 6 해결)
- **D-14:** 선택지 text만 번역. 번호/link 태그는 게임이 후처리. stripQuotes 없음.

### Claude's Discretion
- 연구 결과에 따른 불확실 패치 7개의 포함/제외 최종 결정
- 폰트 로딩 코드의 구체적 간소화 범위
- 내부 코드 구조 (메서드 분리, 변수 네이밍 등)

<specifics>
## Specific Ideas

- v1에서 6개 연쇄 버그의 근본 원인은 "v2 데이터를 v1 가정 코드로 처리"였음. v2 Plugin.cs는 v2 데이터 포맷만 가정해야 함.
- ko_raw/ko_formatted 구분은 export.go에서 이미 해결됨 (ko_formatted 없으면 ko_raw 사용). Plugin.cs에서는 target 필드만 사용.
- 멀티라인 블록 → 개별 줄 분해도 export.go에서 처리됨. Plugin.cs는 단일 줄 매칭만 하면 됨.
- DC/FC 접두사 strip은 export.go에서 body-only entry를 추가하므로, Plugin.cs Stage 2b (DC/FC strip 매칭)는 fallback용.

</specifics>
