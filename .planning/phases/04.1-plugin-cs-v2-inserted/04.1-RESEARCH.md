# Phase 4.1: Plugin.cs v2 재작성 - Research

**Researched:** 2026-03-28
**Domain:** BepInEx/Harmony C# plugin for Unity IL2CPP game modding (translation injection)
**Confidence:** HIGH

## Summary

Plugin.cs v2는 3,083줄 v1 코드를 빈 파일에서 1,500~2,000줄로 새로 작성하는 작업이다. v1 코드를 완전히 분석한 결과, 아키텍처는 명확하다: (1) 데이터 로딩 (translations.json V3 sidecar, textassets, localizationtexts CSV, runtime_lexicon.json), (2) Harmony 패치 등록 (TMP_Text, TextAsset, Dialog, Localization), (3) 폰트 로딩 (TMP_FontAsset fallback injection), (4) TryTranslate 매칭 체인, (5) 캡처/디버깅 인프라.

v1에서 발견된 6개 연쇄 버그의 근본 원인은 모두 "v2 데이터를 v1 가정 코드로 처리"한 것이다. v2 Plugin.cs는 v1 가정(NormalizedMap, StripQuotationMarks, CleanOrphanBoldTags 전체 제거, Bold 폰트 우선, 하드코딩 번역 우선)을 전부 제거하고, v2 데이터 포맷만 가정하는 깨끗한 코드를 작성한다.

**Primary recommendation:** v1 Plugin.cs를 참고 코드로만 사용하고, 빈 파일에서 기능 블록별로 작성. 29건 하드코딩 번역을 runtime_lexicon.json으로 이동하는 것이 첫 번째 작업.

### Pattern 1: V3 Sidecar 전용 로딩
**What:** V3 format object (entries[] + contextual_entries[])만 지원. v1 flat array 호환 제거.
**When to use:** LoadEntriesFromJson
**Example:**
```csharp
// V3 sidecar format only — no v1 flat array fallback
private void LoadEntriesFromJson(string json)
{
    using var doc = JsonDocument.Parse(json);
    var root = doc.RootElement;

    if (root.ValueKind != JsonValueKind.Object ||
        !root.TryGetProperty("format", out var fmt) ||
        fmt.GetString() != "esoteric-ebb-sidecar.v3")
    {
        Log.LogError("translations.json: expected v3 sidecar format");
        return;
    }

    // entries[] -> TranslationMap (exact match, no NormalizedMap)
    foreach (var item in root.GetProperty("entries").EnumerateArray())
    {
        var source = ReadString(item, "source");
        var target = ReadString(item, "target");
        if (!string.IsNullOrEmpty(source) && !string.IsNullOrEmpty(target))
            TranslationMap[source] = target;
    }

    // contextual_entries[] -> ContextualMap
    if (root.TryGetProperty("contextual_entries", out var ctx))
    {
        foreach (var item in ctx.EnumerateArray())
            AddContextualEntry(item);
    }
}
```

### Pattern 2: TryTranslate 5단계 체인 (D-05)
**What:** GeneratedPattern -> TranslationMap -> DC/FC strip -> Contextual -> RuntimeLexicon
**Note:** CONTEXT.md D-05 순서를 정확히 따름. GeneratedPattern이 먼저인 이유는 동적 UI 패턴("Ability Scores - Remaining Points: N")이 TranslationMap에 없기 때문.
```csharp
internal static bool TryTranslate(ref string value, string origin = "unknown")
{
    if (string.IsNullOrEmpty(value)) return false;

    var original = value;

    // Stage 1: Generated patterns (dynamic UI — not in TranslationMap)
    if (TryTranslateGeneratedPattern(ref value)) { ... return true; }

    // Stage 2: Exact dictionary lookup
    if (TranslationMap.TryGetValue(value, out var t) && !string.IsNullOrEmpty(t))
    { value = t; ... return true; }

    // Stage 3: DC/FC prefix strip -> lookup body
    if (TryTranslateDcFcStrip(ref value, original)) { return true; }

    // Stage 4: Contextual (source_file + history disambiguation)
    if (TryTranslateContextual(ref value, original)) { return true; }

    // Stage 5: Runtime lexicon (exact + substring + regex)
    if (TryTranslateRuntimeLexicon(ref value)) { return true; }

    RecordMiss(original);
    CaptureUntranslated(original);
    return false;
}
```

### Pattern 3: runtime_lexicon.json v2 (exact 타입 추가)
**What:** v1은 substring + regex만 지원. v2는 exact (전체 문자열 매칭) 타입 추가.
**Why:** MenuDirectOverrides 8건, StatNameOverrides 6건 등 하드코딩 번역을 외부 파일로 이동.
```json
{
  "format": "esoteric-ebb-runtime-lexicon.v2",
  "exact_replacements": [
    { "find": "New Game", "replace": "새 게임" },
    { "find": "Load", "replace": "불러오기" },
    { "find": "Strength", "replace": "힘" }
  ],
  "substring_replacements": [
    { "find": "Gained experience: ", "replace": "경험치 획득: " }
  ],
  "regex_rules": [
    { "name": "ability_remaining", "pattern": "^Ability Scores - Remaining Points: (\\d+)$", "replace": "능력치 - 남은 포인트: $1" }
  ]
}
```

### Anti-Patterns to Avoid (v1 버그에서 학습)
- **NormalizedMap 폴백:** v2 데이터는 exact match 전용. normalize는 false positive 유발.
- **하드코딩 번역:** Plugin.cs 안에 번역 문자열 0건. 전부 runtime_lexicon.json.
- **StripQuotationMarks:** 선택지 따옴표를 제거하면 게임의 번호 매칭 실패 (Bug 6).
- **CleanOrphanBoldTags 전체 제거:** v2 번역에 의도적 <b> 태그 있음. orphan-only 제거만 허용 (Bug 4).
- **Bold 폰트 우선:** 한글 vs 영문 글자 두께 불일치 원인. Regular 우선 (폰트 weight 불일치 이슈).
- **Stage 1 우선순위:** 하드코딩 번역이 DB 번역을 덮어쓰는 문제. runtime_lexicon은 최하위.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| JSON 파싱 | 수동 문자열 파싱 | System.Text.Json JsonDocument | .NET 6 built-in, 스트리밍 파서, 메모리 효율적 |
| Harmony 패치 | 수동 IL rewriting | HarmonyLib prefix/postfix | BepInEx 번들, 안정적 API |
| 폰트 생성 | 직접 glyph 렌더링 | TMP_FontAsset.CreateFontAsset + fallback chain | Unity TMP 표준 API |
| CSV 파싱 | Split(',') | ParseCsvLine (v1에서 검증됨) | 따옴표 내부 쉼표 처리 필요 |
| 재진입 방지 | bool 플래그 | [ThreadStatic] depth counter + EnterPatchedCall/ExitPatchedCall | TMP setter -> TryTranslate -> text 변경 -> 재귀 호출 방지 |

## Common Pitfalls

### Pitfall 1: 재진입 무한 루프
**What goes wrong:** TMP_Text.text setter 패치가 TryTranslate를 호출하고, 번역 결과를 다시 text에 설정하면 setter가 재호출됨.
**Why it happens:** Harmony prefix가 setter를 가로채는데, setter 내부에서 다시 text를 설정하면 재귀 발생.
**How to avoid:** [ThreadStatic] _patchReentryDepth + EnterPatchedCall()/ExitPatchedCall() 패턴. v1에서 이미 구현됨 -- 그대로 가져오기.
**Warning signs:** 게임 프리즈 또는 StackOverflowException.

### Pitfall 2: Assembly 로딩 타이밍
**What goes wrong:** BepInEx Load() 시점에 TMPro/Assembly-CSharp가 아직 로딩되지 않아 패치 실패.
**Why it happens:** IL2CPP 게임은 어셈블리가 지연 로딩됨.
**How to avoid:** v1의 OnAssemblyLoad 이벤트 + TryApplyPatches 재시도 패턴 유지. 각 패치에 _xxxPatched 플래그로 중복 방지.
**Warning signs:** "Patch deferred" 로그가 영구적으로 남음 (resolved 없음).

### Pitfall 3: TextAsset override 이름 매칭
**What goes wrong:** TextAsset 이름이 대소문자 불일치로 매칭 실패.
**Why it happens:** Unity가 에셋 이름을 반환하는 방식과 파일명이 다를 수 있음.
**How to avoid:** TextAssetOverrides 딕셔너리에 StringComparer.OrdinalIgnoreCase 사용 (v1에서 이미 적용).
**Warning signs:** text_asset_override_miss_logged 카운터 증가.

### Pitfall 4: ContextualMap 메모리 사용량
**What goes wrong:** 35,029 contextual entries를 모두 메모리에 로딩하면 GC 압박.
**Why it happens:** 각 ContextualEntry가 8개 string 필드를 가짐.
**How to avoid:** ContextualEntry에서 v2에서 사용하지 않는 필드 제거 (ContextEn, TranslationLane은 v2 TryTranslateContextual에서 사용하지 않음 -- 확인 필요). 최소한 string interning 고려.
**Warning signs:** 게임 시작 시 로딩 시간 증가 (>3초).

### Pitfall 5: DC/FC prefix regex 불일치
**What goes wrong:** Plugin.cs의 DC/FC regex와 export.go의 dcfcPrefixRe가 다르면 매칭 실패.
**Why it happens:** 두 곳에 동일한 regex가 하드코딩됨.
**How to avoid:** 정확히 동일한 패턴 사용: `^[A-Z]{2}\d+\s+\w+-`. export.go에서 body-only entry도 추가하므로 대부분 TranslationMap에서 직접 매칭되지만, fallback으로 strip 매칭도 유지.
**Warning signs:** DC/FC 선택지가 원문 노출.

### Pitfall 6: 빌드 시 BepInExRoot 경로
**What goes wrong:** .csproj 기본 BepInExRoot가 `C:\Program Files (x86)\Steam\...`으로 설정되어 있어 빌드 실패.
**Why it happens:** 실제 게임은 E: 드라이브에 설치됨.
**How to avoid:** `dotnet build /p:BepInExRoot="E:\SteamLibrary\steamapps\common\Esoteric Ebb\BepInEx"` 사용.
**Warning signs:** 빌드 에러 "Reference not found: BepInEx.Core".

```csharp
[ThreadStatic] private static int _patchReentryDepth;

private static bool EnterPatchedCall()
{
    if (_patchReentryDepth > 0) return false;
    _patchReentryDepth++;
    return true;
}

private static void ExitPatchedCall()
{
    if (_patchReentryDepth > 0) _patchReentryDepth--;
}
```

```csharp
// v2: only strip unmatched <b>/<\/b> tags, preserve balanced pairs
private static string CleanOrphanBoldTags(string text)
{
    if (string.IsNullOrEmpty(text)) return text;

    int opens = 0, closes = 0;
    int idx = 0;
    while ((idx = text.IndexOf("<b>", idx, StringComparison.Ordinal)) >= 0) { opens++; idx += 3; }
    idx = 0;
    while ((idx = text.IndexOf("</b>", idx, StringComparison.Ordinal)) >= 0) { closes++; idx += 4; }

    if (opens == closes) return text; // balanced -- keep

    // Unbalanced: strip all (safety)
    return text.Replace("<b>", "").Replace("</b>", "");
}
```

## Open Questions

1. **ContextualMap의 ContextEn 필드 활용도**
   - What we know: v1에서 히스토리 매칭에 사용. runtime state에서 contextual_hits=69, source_file_context_hits=53.
   - What's unclear: ContextEn이 없으면 몇 건이 miss되는지.
   - Recommendation: ContextEn 유지하되, V3 sidecar에 context_en 필드가 있는지 확인 필요. 현재 V3Entry에는 context_en이 없으므로, ContextualMap에서 ContextEn 기반 히스토리 매칭은 동작하지 않을 것. source_file 매칭만으로 충분한지 확인 (53/69 = 77% 커버).

2. **GeneratedPattern의 regex_rules 이동 범위**
   - What we know: v1 TryTranslateGeneratedPattern에 "Ability Scores" 패턴 1개 + TranslateShortSystemPhrase에 regex 패턴들.
   - What's unclear: GeneratedPattern을 완전히 runtime_lexicon으로 대체할 수 있는지. "Ability Scores - Remaining Points: N"은 런타임에 숫자가 변하는 동적 패턴.
   - Recommendation: regex_rules로 이동 가능. TryTranslateGeneratedPattern 메서드 자체를 제거하고 RuntimeLexicon의 regex_rules에서 처리. 그러면 TryTranslate 체인이 4단계로 더 단순화됨: TranslationMap -> DC/FC strip -> Contextual -> RuntimeLexicon.

3. **히트율 개선 범위**
   - What we know: 현재 hit=687, miss=8,552. 하지만 이것은 v1 기준이고, TextAsset override가 18,423 블록을 처리하므로 TryTranslate 미스 대부분은 이미 번역된 텍스트의 중복 조회일 수 있음.
   - What's unclear: v2 Plugin.cs 후 실제 미번역 건수.
   - Recommendation: v2 배포 후 untranslated_capture.json 재수집으로 판단. 95%+ 커버리지는 인게임 검증으로 확인.
